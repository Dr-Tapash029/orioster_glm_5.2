    17вҶ’## 1. PRODUCT IDENTITY
    18вҶ’
    19вҶ’**ORIOSTER** IS AN **OFFLINE-FIRST, AI-ASSISTED, MULTI-TENANT HOSPITAL MANAGEMENT SYSTEM (HMS)** ENGINEERED
    20вҶ’FOR LOW-RESOURCE HEALTHCARE ENVIRONMENTS. IT OPERATES AS A LOCAL-FIRST CLINICAL OPERATING SYSTEM WITH
    21вҶ’ASYNCHRONOUS AI AUGMENTATION вҖ” NOT A CLOUD-DEPENDENT SAAS TOOL.
    22вҶ’
    23вҶ’> **CORE DESIGN PHILOSOPHY:**
    24вҶ’> OFFLINE-FIRST В· CACHE-FIRST В· HUMAN-CONTROLLED В· AI IS ASSISTIVE, NEVER AUTHORITATIVE.

    51вҶ’## 4. IN-SCOPE FEATURES
    52вҶ’
    53вҶ’### CORE CLINICAL WORKFLOWS
    54вҶ’- 10-STEP PATIENT ENTRY WIZARD (PATIENTENTRYWIZARD)
    55вҶ’- VITALS RECORDING WITH LOCAL TRIAGE LOGIC (NO AI REQUIRED)
    56вҶ’- CHIEF COMPLAINT CAPTURE (STRUCTURED DROPDOWN + OPTIONAL FREE-TEXT)
    57вҶ’- PAST HISTORY TAGGING (STRUCTURED TAGS + CAPPED NOTES)
    58вҶ’- ONGOING MEDICATION TRACKING WITH DUPLICATE AND INTERACTION WARNINGS
    59вҶ’- HYPERSENSITIVITY / ALLERGY RECORDING WITH SEVERITY TAGGING
    60вҶ’- APPOINTMENT SCHEDULING AND DOCTOR ASSIGNMENT WITH TIME SLOTS
    61вҶ’- ROLE-BASED DASHBOARDS (DOCTOR / NURSE / ADMIN / LAB TECHNICIAN)
    62вҶ’
    63вҶ’### AI-ASSISTED MODULES (ORIO AI)
    64вҶ’- DIFFERENTIAL DIAGNOSIS вҖ” 3 MOST-PROBABLE DIAGNOSES WITH CONFIDENCE METER
    65вҶ’- TREATMENT PLAN GENERATION (POST DOCTOR-CONFIRMED DIAGNOSIS)
    66вҶ’- PRESCRIPTION GENERATION (DOCTOR-CONFIRMED, CUSTOMIZABLE, EXPORT-READY)
    67вҶ’- LAB REPORT ANALYSIS вҖ” PARAMETRIC ENTRY + AI CLINICAL FEEDBACK ADVISORY
    68вҶ’- INVOICE GENERATION вҖ” AI-FILLED, FIXED TEMPLATE, ITEMIZED
    69вҶ’- AI HUB вҖ” CENTRAL COMMAND MODULE FOR INVOICE, LABS, PRESCRIPTION, MEDICAL CERTIFICATE
    70вҶ’
    71вҶ’### INFRASTRUCTURE
    72вҶ’- OFFLINE-FIRST SYNC VIA POWERSYNC SDK (BACKGROUND, INVISIBLE TO USER)
    73вҶ’- ROLE-BASED ACCESS CONTROL (RBAC) ENFORCED AT RLS + ROUTER LEVEL
    74вҶ’- END-TO-END ENCRYPTION (E2EE) ON ALL CLINICAL DATA VIA AES-GCM
    75вҶ’- MULTI-TENANT ARCHITECTURE вҖ” DATA PARTITIONED PER FACILITY AND ROLE
    76вҶ’- SQLCIPHER AES-256 ENCRYPTION AT REST

    93вҶ’## 6. PRIVACY GUARANTEES (NON-NEGOTIABLE)
    94вҶ’
    95вҶ’1. **NO RAW PHI LEAVES THE DEVICE** вҖ” EVER, UNDER ANY CIRCUMSTANCE, FOR ANY REASON.
    96вҶ’2. **AI RECEIVES ONLY `PATIENT_SUMMARY_V1`** вҖ” A DE-IDENTIFIED, COMPRESSED (вүҘ70%) SUMMARY
    97вҶ’   GENERATED FULLY LOCALLY AT STEP 8.
    98вҶ’3. **AI CALLS ARE HARD-DISABLED UNTIL STEP 8 IS COMPLETE** вҖ” ENFORCED BY A CONTROLLER-LEVEL GUARD
    99вҶ’   THAT CANNOT BE BYPASSED FROM ANY UI LAYER.
   100вҶ’4. **ALL CLINICAL DATA IS AES-256 ENCRYPTED AT REST** (SQLCIPHER) AND AES-GCM ENCRYPTED
   101вҶ’   BEFORE SYNC (E2EE VIA DART CRYPTOGRAPHY PACKAGE).
   102вҶ’5. **AI LOGS ARE ENCRYPTED, SCOPED PER USER, AND PHI-SAFE** BEFORE WRITING TO ANY LOG SINK.

   157вҶ’## 10. HERMES AGENT вҖ” ORCHESTRATION RULES
   158вҶ’
   159вҶ’THESE RULES GOVERN HOW HERMES AGENT MANAGES THE BUILD LIFECYCLE ACROSS SESSIONS.
   160вҶ’
   161вҶ’### 10.1 PRE-BUILD CHECKLIST (EVERY SESSION)
   162вҶ’BEFORE WRITING ANY CODE, HERMES AGENT MUST:
   163вҶ’1. INVOKE `AGENT-ORCHESTRATOR` вҶ’ READ `PROGRESS_TRACKER.MD` TO IDENTIFY CURRENT KANBAN COLUMN
   164вҶ’2. INVOKE `CODE-WORKFLOW` вҶ’ READ `MEMORY.MD` IF IT EXISTS (PRIOR SESSION STATE)
   165вҶ’3. READ `ARCHITECTURE.MMD`, `FOLDER_STRUCTURE.MD`, AND `CODE_STANDARDS.MD`
   166вҶ’4. READ `UI_REGISTRY.MD` BEFORE WRITING ANY UI COMPONENT
   167вҶ’5. READ `UI_TOKENS.MD` BEFORE WRITING ANY WIDGET CODE
   168вҶ’6. ROUTE THE CURRENT TASK TO THE CORRECT PRIMARY SKILL PER В§9 ABOVE
   169вҶ’
   170вҶ’### 10.2 DURING BUILD
   171вҶ’- `KANBAN-WORKER` UPDATES THE TASK STATUS IN `PROGRESS_TRACKER.MD` AS WORK PROGRESSES
   172вҶ’- `CLAUDE-CODE` WRITES CODE; `FLUTTER-DEV` ENFORCES PATTERNS; `CODE-WORKFLOW` MAINTAINS FILE ORDER
   173вҶ’- `AGENT-ORCHESTRATOR` COORDINATES IF MULTIPLE DOMAINS ARE TOUCHED IN ONE TASK
   174вҶ’- `SUBAGENT-DRIVEN-DEVELOPMENT` HANDLES TASKS THAT REQUIRE PARALLELISM ACROSS FEATURES
   175вҶ’
   176вҶ’### 10.3 AFTER EACH FEATURE
   177вҶ’1. `TEST-DRIVEN-DEVELOPMENT` вҶ’ WRITE REQUIRED WIDGET AND UNIT TESTS
   178вҶ’2. `UI-UX-PRO-MAX` + `CLAUDE-DESIGN` вҶ’ RUN `/IMPRINT` ON ANY NEW UI COMPONENT
   179вҶ’3. `CODE-WORKFLOW` вҶ’ RUN `/REVIEW` AGAINST `CODE_STANDARDS.MD` AND `ARCHITECTURE.MMD`
   180вҶ’4. `KANBAN-ORCHESTRATOR` вҶ’ MOVE TASK TO DONE COLUMN IN `PROGRESS_TRACKER.MD`
   181вҶ’5. `CODE-WORKFLOW` вҶ’ RUN `/REMEMBER SAVE` TO WRITE `MEMORY.MD`
   182вҶ’
   183вҶ’### 10.4 FAILURE HANDLING
   184вҶ’- IF A BUILD FAILS, INVOKE `RECOVER` SKILL BEFORE CONTINUING
   185вҶ’- IF SESSION CONTEXT IS POLLUTED, `KANBAN-ORCHESTRATOR` LOGS A RESET NOTE AND STARTS FRESH
   186вҶ’- `AGENT-ORCHESTRATOR` PREVENTS CONCURRENT BUILDS THAT TOUCH THE SAME CONTROLLER OR SERVICE FILE
